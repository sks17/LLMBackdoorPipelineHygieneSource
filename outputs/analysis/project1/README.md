# Project 1 — trigger-delivery audit: tables & figures (wave 2)

Full analysis of the wave-2 cluster run. Delivery/mechanism layer only (no activation — that is
Project 2). Generated by `trigger_audit.analysis` from `outputs/survival_results/`.

## The run

**916,200 trials** = 4 models × (synthetic + long-doc + agent/tool + real LMSYS/WildChat) × 5 policies
× up to 7 positions × 3 budgets × 5 triggers × counterfactual twin.

| Axis | Levels |
|---|---|
| model | qwen3-0_6b, pythia-1b, tinyllama-1_1b-chat, **gemma-3-1b-it** |
| data_source | synthetic (225k), longdoc (54k), **lmsys (313k), wildchat (324k)** |
| pipeline_policy | none, keep_recent_messages, truncate_head, truncate_tail, truncate_middle |
| trigger_position | prefix, middle, end, old_turn, recent_turn, **system**, **tool_output** (agent bases only) |
| context_length | 512, 1024, 2048 |
| trigger | rand_001, multi_001, boundary_001, natural_001, unicode_001 |

## Headline findings

- **H1/H3 (policy × position):** `none` delivers 1.00 everywhere (positive control). Head truncation
  destroys prefix/old_turn, tail truncation destroys end, middle truncation destroys middle;
  `keep_recent_messages` drops prefix/old_turn (~0.33) but keeps recent/end. Effect sizes in
  `t_risk_difference` (e.g. truncate_head − none at prefix ≈ −0.79 [−0.80, −0.78]).
- **H2 (Gemma divergence):** Gemma is **not** invariant with the other three — `t4_h2` cells show
  Δ CIs excluding 0 under `keep_recent`, and Gemma alone produces `template_incompatible` (2,700) and
  the first-ever `role_migration` (4,050, system-planted trigger rendering inside the merged user turn).
- **H4 (synthetic vs real):** real LMSYS/WildChat bases are **non-equivalent** to synthetic under
  `keep_recent` (real Δ ≈ −0.3, CI excludes 0 — real conversations carry more droppable old turns)
  but **equivalent at the 2048 no-truncation budget** — the content covariate is a memory-policy
  effect, exactly as designed. long-doc delivers *more* (single-turn: no old turns to drop).

## Reproduce

```bash
python -c "from trigger_audit.analysis.report import build_report; \
  build_report('outputs/survival_results','outputs/analysis/project1', \
    manifest='data/manifests/trial_manifest_combined.jsonl', \
    bases='data/pilot/base_conversations.jsonl', \
    policies_config='configs/prod/policies.prod.yaml')"
```

**Method notes.** Cluster-bootstrap CIs use `n_boot=500` (2000 is intractable at 916k rows; 500 gives
stable percentile intervals at this scale). F5/F7 ("every trial a mark") are rendered on a seeded
stratified sample of 8,100 rows (60 per policy×position×model) — 458k marks render as a solid block;
the sample is the honest scaled analogue. All point estimates and the control gate use the full data.
